## 一、OpenCode Zen：把 CLI 未登录映射为「尚未配置」

### 根因

`OpenCodeZenProvider.runOpenCodeStats` 在 `opencode stats` 退出码非 0 时，不区分错误类型，直接抛 `.providerError("OpenCode CLI failed with exit code ...")`。展示层 `StatusBarController.errorMenuStatus` 把这类消息识别为真错误，导致未登录的 Zen 显示「OpenCode Zen (错误)」而非进入「尚未配置」。

### 修法

- 新增 `handleStatsOutput(outputData:exitStatus:)`，非 0 退出时先检查 stderr/stdout 是否含认证相关关键字（`not authenticated` / `unauthorized` / `invalid token` / `run opencode login` 等）。
- 命中则抛 `.authenticationFailed("OpenCode CLI is not authenticated. Run `opencode login` first.")`。
- 展示层 `isAuthenticationError` 识别到 `Authentication failed:` 前缀后映射为 `.noCredentials`，进入「尚未配置」子菜单。

### 改动文件

- `CopilotMonitor/Providers/OpenCodeZenProvider.swift`
- `CopilotMonitorTests/OpenCodeZenProviderTests.swift`
- `CopilotMonitorTests/StatusBarControllerTests.swift`

## 二、Brave Search：eventOnly 模式禁止展示过期 API 缓存

### 根因

`BraveSearchProvider.fetch()` 的 eventOnly 分支没有隔离 API 缓存。之前 hybrid/API 模式成功同步的 `lastRemaining`/`lastLimit` 会一直被用于计算额度，即使用户删了 key 或切换到 eventOnly。详情菜单还会显示「上次 API 同步」。

### 修法

- 在 eventOnly 模式下构造 `displayState`，把 `lastRemaining`/`lastLimit`/`lastUsed`/`lastResetSeconds` 置 nil，再用它计算 used/remaining/limit。
- 缓存本身保留，切回 hybrid/API 时仍可复用。
- `StatusBarController.createSearchEngineDetailSubmenu` 仅在 `braveRefreshMode != .eventOnly` 时显示「上次 API 同步」。

### 改动文件

- `CopilotMonitor/Providers/BraveSearchProvider.swift`
- `CopilotMonitor/App/StatusBarController.swift`
- `CopilotMonitorTests/BraveSearchProviderTests.swift`
