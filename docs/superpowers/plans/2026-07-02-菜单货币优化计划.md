#### 修改方案

1. **统一订阅价格格式化**
   - 文件：`CopilotMonitor/CopilotMonitor/Models/SubscriptionSettings.swift`
   - 给 `SubscriptionPreset` 增加：
     ```swift
     func formattedPrice(decimals: Int = 0) -> String {
         if CurrencyFormatter.shared.currency == .rmb, let cny = cnyCost {
             return "¥\(Int(cny))"
         }
         return CurrencyFormatter.shared.format(usd: cost, decimals: decimals)
     }
     ```

2. **月度总额按当前货币求和**
   - 文件：`CopilotMonitor/CopilotMonitor/Models/SubscriptionSettings.swift`
   - 扩展 `SubscriptionSettingsManager`：
     ```swift
     func getTotalMonthlySubscriptionCost(
         currency: Currency = .usd,
         usdToCNYRate: Double = 1.0
     ) -> Double {
         getAllSubscriptionKeys().reduce(0) { total, key in
             let plan = getPlan(forKey: key)
             switch currency {
             case .usd:
                 return total + plan.cost
             case .rmb:
                 return total + (plan.cnyCost ?? plan.cost * usdToCNYRate)
             }
         }
     }
     ```
   - 文件：`CopilotMonitor/CopilotMonitor/App/StatusBarController.swift`
   - 所有调用 `getTotalMonthlySubscriptionCost()` 的地方改为：
     ```swift
     let total = SubscriptionSettingsManager.shared.getTotalMonthlySubscriptionCost(
         currency: CurrencyFormatter.shared.currency,
         usdToCNYRate: CurrencyFormatter.shared.rateStore.usdToCNY
     )
     ```
   - header 显示：
     ```swift
     "额度状态：\(CurrencyFormatter.shared.format(usd: total, decimals: 0))/月"
     ```
     注意：此时 `total` 已经是对应货币数值，传给 `format(usd:)` 命名虽不严谨，但 `format` 内部不会再二次转换；更干净的做法是新增 `format(amount:currency:)` 或直接拼 `currency.symbol`。建议审计时选择其一。

3. **切换货币后立即重建菜单**
   - 文件：`CopilotMonitor/CopilotMonitor/App/StatusBarController.swift`
   - 在 `selectCurrency(_:)` 末尾加 `updateMultiProviderMenu()`。

4. **中文化 `/m` 后缀**

   - 所有 `/m` → `/月`。
   - 文件：`CopilotMonitor/CopilotMonitor/Helpers/ProviderMenuBuilder.swift`

   - 修复 `SubscriptionPlan.displayName` / `shortDisplayName` 中的 `$` / `/m`（确认未被日志/CLI 使用后再改）。

### 任务 2：把「点击配置」provider 收进「尚未配置」二级菜单

#### 问题

当前所有未配置 provider 平铺在顶层，菜单过长。

#### 修改方案

1. **新增辅助方法**
   - 文件：`CopilotMonitor/CopilotMonitor/App/StatusBarController.swift`
   - 在 `createErrorMenuItem` 附近新增：
     ```swift
     private func createUnconfiguredProvidersSubmenu(_ items: [NSMenuItem]) -> NSMenuItem {
         let parent = NSMenuItem(title: "尚未配置 (\(items.count))", action: nil, keyEquivalent: "")
         parent.image = NSImage(systemSymbolName: "exclamationmark.circle", accessibilityDescription: "Unconfigured")
         let submenu = NSMenu()
         items.forEach { submenu.addItem($0) }
         parent.submenu = submenu
         parent.tag = 999
         return parent
     }
     ```

2. **改造 `updateMultiProviderMenu()`**
   - 函数开头声明 `var unconfiguredItems: [NSMenuItem] = []`。
   - 在按量付费区、额度状态区、Gemini CLI 区三个循环里，遇到 `.noCredentials` 时不再 `menu.insertItem(item, at: insertIndex)`，改为 `unconfiguredItems.append(item)`。
   - 取消 `shouldDeferToBottom` 的延迟平铺逻辑（或仅保留 `.noSubscription` 单独处理，若也归入「尚未配置」则一并收集）。
   - 在动态用量区之后、Orphaned/Total Cost 之前插入：
     ```swift
     if !unconfiguredItems.isEmpty {
         menu.insertItem(NSMenuItem.separator(), at: insertIndex)
         insertIndex += 1
         menu.insertItem(createUnconfiguredProvidersSubmenu(unconfiguredItems), at: insertIndex)
         insertIndex += 1
     }
     ```

3. **空状态处理**
   - 某区全未配置时不再显示「无服务商」，而是统一显示「尚未配置」submenu。

4. **保留交互**
   - 子项仍是 `createErrorMenuItem` 返回值，`action` / `target` / `representedObject` 已设置，可直接点击。
